## Common Commands

```bash
cd agents/the-{name}
poetry install
poetry run python -m src.setup     # Register on Nevermined (one-time)
poetry run python -m src.server    # Start MCP server
```

All agents use `package-mode = false` in pyproject.toml. Always use `poetry run python -m src.<module>`.

## Agent Service Pattern

Every agent follows this structure:

```
agents/the-{name}/
├── src/
│   ├── __init__.py
│   ├── server.py          # MCP server (entry point)
│   ├── setup.py           # Nevermined registration
│   └── {module}.py        # Core logic (discovery.py, reputation.py, qa.py, etc.)
├── .env.example
├── pyproject.toml
├── Procfile               # Railway: web: python -m src.server
├── requirements.txt       # Railway deps
└── README.md
```

MCP server pattern:
```python
from payments_py.mcp import PaymentsMCP

mcp = PaymentsMCP(payments, name="service-name", agent_id=AGENT_ID, ...)

@mcp.tool(credits=0)  # FREE during promotional period
def my_tool(param: str) -> str:
    """What it does. What it returns. When to use. Honest limitations."""
    return result
```

Setup registration pattern:
```python
from payments_py.plans import get_free_price_config, get_dynamic_credits_config

price_config = get_free_price_config()
credits_config = get_dynamic_credits_config(credits_granted=100, min_credits_per_request=0, max_credits_per_request=0)
result = payments.agents.register_agent_and_plan(...)
```

## Environment Variables

Each agent has `.env.example`. Required:

```bash

# For The Architect + The Gold Star:

# For Railway deployment:

```

## Agent Discovery (The Ledger)

The Ledger serves agent-friendly discovery endpoints:

- `agenteconomy.io/llms.txt` -- Plain text service manifest for LLMs
- `agenteconomy.io/.well-known/agent.json` -- A2A agent card with all services
- `agenteconomy.io/api/sellers` -- JSON marketplace data
- `agenteconomy.io/api/buyers` -- JSON buyer data
- `agenteconomy.io/api/analysis` -- Marketplace analysis
- Any 404 returns JSON with hints and available endpoints

## Railway Deployment

- **Builder:** Railpack (auto-detects from requirements.txt + Procfile)
- **Deploy:** `cd /repo/root && railway up -s <service-name> --detach`
- **Env vars:** `railway variables --set "KEY=value" -s <service-name>`
- **Logs:** `railway logs -s <service-name>`
- Each service's `rootDirectory` is set in Railway dashboard

## Hackathon Discovery Directory

Agents are registered in the hackathon marketplace at `nevermined.ai/hackathon/register`.

API: `https://nevermined.ai/hackathon/register/api/`
- `GET /agents` (with `x-nvm-api-key` header) -- our registered agents
- `GET /discover?side=sell` -- public marketplace
- `PATCH /agents/{id}` -- update agent metadata

To appear in marketplace, agents need: `category`, `servicesSold`, `servicesProvidedPerRequest`.

## Key Design Principles

1. **All tools FREE** -- credits=0 across the board during promotional period
2. **Honest descriptions** -- every tool lists capabilities AND limitations
3. **Agent-friendly** -- JSON errors with hints, llms.txt, agent.json, normalized data
4. **Independent services** -- each agent is a standalone Poetry project
5. **Same pattern** -- all MCP servers follow identical structure
